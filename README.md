Модуль «Учет финансов» (Python) 

Мини‑консольное приложение для учета доходов/расходов: 

Доменные модели: BankAccount, Category, Operation. 

Фасады для CRUD и сценариев, импортеры JSON/CSV/YAML. 

Паттерны: Facade, Command + Decorator (timing), Template Method (import), Factory (через фасады). 

SOLID/GRASP: высокая связность внутри модулей, низкая связность между ними, явные интерфейсы. 

Возможности: 

Создание/редактирование/удаление счетов, категорий, добавление операций. 

Обновление баланса по операциям (знак суммы валидируется по типу категории). 

Импорт данных из JSON/CSV/YAML (Template Method). 

Измерение времени пользовательских сценариев (Decorator над командами). 

Пример аналитики (можно расширять): баланс, группировки по категориям/типам. 

  

Архитектура и паттерны 

SOLID/GRASP 

SRP: модели не знают о вводе/выводе; фасады отвечают за сценарии. 

OCP: новые импортеры добавляются без изменения существующего кода. 

LSP: импортеры следуют базовому интерфейсу DataImporter. 

ISP: узкие интерфейсы (фасады, импортеры) без лишних методов. 

DIP: код зависит от абстракций (DataImporter), а не от конкретных форматов. 

High Cohesion: у пакетов чёткая зона ответственности. 

Low Coupling: минимальные зависимости между модулями через публичные интерфейсы.
